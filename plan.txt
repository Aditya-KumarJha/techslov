# Social Media Video Comparison RAG Assistant

## Objective

Build a full-stack RAG chatbot that accepts:

* YouTube Video URL
* Instagram Reel URL

and allows creators to compare performance, engagement, hooks, content style, and creator information through a conversational interface.

---

# Data Ingestion Pipeline

## Step 1: Receive URLs

Input:

Video A URL
Video B URL

Supported:

* YouTube
* Instagram Reels

---

## Step 2: Metadata Extraction

For each video collect:

* Video ID
* Platform
* Title
* Creator Name
* Creator Handle
* Follower Count
* Views
* Likes
* Comments
* Upload Date
* Duration
* Hashtags

---

## Step 3: Engagement Calculation

Formula:

Engagement Rate =
(Likes + Comments) / Views × 100

Store result as metadata.

---

## Step 4: Transcript Extraction

### Primary Strategy

Use native platform captions whenever available.

Advantages:

* Faster
* More accurate
* No compute cost
* Already timestamped

---

### Fallback Strategy

If captions unavailable:

Download audio

Run local Whisper transcription

Generate timestamped segments

Advantages:

* Works for any video
* No paid transcription API

---

# Transcript Processing

## Transcript Format

Each segment:

* Text
* Start Time
* End Time
* Video ID

Example Metadata

videoId: A
start: 0
end: 5
text: "..."

---

## Chunking Strategy

Chunk Size:

* 400–600 tokens

Overlap:

* 50–100 tokens

Metadata Attached:

* videoId
* chunkId
* startTime
* endTime
* creatorName
* platform

---

Embedding Vector

# Retrieval Architecture

When a question arrives:

Example:

"Why did Video A get more engagement than Video B?"

System retrieves:

* Relevant chunks from Video A
* Relevant chunks from Video B
* Engagement metadata
* Creator metadata

Context assembled before LLM call.

---

# Chat Agent

## Memory

Maintain:

* Previous Questions
* Previous Answers
* Comparison Context

Example:

User:
"Who is creator of Video B?"

Follow-up:

"What improvements can they make?"

Agent remembers Video B context automatically.

---

# Source Citation System

Every response includes:

Video Source
Chunk ID
Timestamp Range

Example:

Source:
Video A
Chunk 12
00:00–00:18

This satisfies explainability requirements.

---

# Scalability Strategy

## Cost Optimization

Use native captions whenever possible.

Only invoke Whisper when captions unavailable.

This significantly reduces compute costs.

---

## Embedding Optimization

Generate embeddings once.

Cache permanently.

Avoid re-embedding identical videos.

---

## Metadata Caching

Store:

* Video metadata
* Engagement metrics
* Transcript

Reuse on future requests.

---

## LLM Optimization

Use retrieval first.

Send only relevant chunks.

Reduce token consumption.
